## Notes for SOP Authors

### Using This Template
1. Replace all bracketed placeholders with actual content
2. Remove sections that don't apply to your specific SOP
3. Add additional steps as needed for your process
4. Ensure each major step has clear Purpose/Rationale
5. Include specific Success/Failure criteria for outcomes
6. Create comprehensive quality gates for validation
7. **Include real-world examples when they exist** - they make SOPs dramatically more practical
8. Reference SOP-C011 as an excellent example of this template in use

### Structure Guidelines
- **Major Steps**: Use numbered steps (Step 1, Step 2, etc.)
- **Sub-steps**: Use letter notation (Step 1.a, Step 1.b, etc.)
- **Code blocks**: Include language specification for syntax highlighting
- **Outcomes**: Always include both Success and Failure scenarios
- **Quality Gates**: Split into Pre- and Post- process validation
- **Examples**: Link to real-world examples when available - avoid embedding to prevent duplication
- **Cross-references**: Link to relevant SOPs for additional guidance
